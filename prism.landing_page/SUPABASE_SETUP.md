# Supabase setup cho lưu kết quả bài Test

Trang `Test` hiện đã tự động lưu dữ liệu sau khi người dùng hoàn thành bài test vào bảng `test_submissions`.

## 1) Tạo bảng trong Supabase

Vào **Supabase Dashboard** → **SQL Editor** → chạy script sau:

```sql
create table if not exists public.test_submissions (
  id bigint generated by default as identity primary key,
  name text not null,
  age smallint not null check (age >= 1 and age <= 120),
  email text not null,
  score smallint not null check (score >= 0 and score <= 30),
  result_class text not null check (result_class in ('low', 'mid', 'high')),
  result_title text not null,
  created_at timestamptz not null default now()
);

alter table public.test_submissions enable row level security;

grant usage on schema public to anon, authenticated;
grant insert on table public.test_submissions to anon, authenticated;

drop policy if exists "allow_anonymous_insert_test_submissions" on public.test_submissions;

create policy "allow_anonymous_insert_test_submissions"
on public.test_submissions
for insert
to anon, authenticated
with check (true);
```

## 1.1) Tạo bảng Blog (category enum: low/mid/high)

Nếu bạn muốn thêm bảng `Blog` với các cột `title`, `description`, `ImageUrl`, `ReferenceUrl` và `category` dạng string, chạy script sau trong **SQL Editor**:

```sql
create table if not exists public."Blog" (
  id bigint generated by default as identity primary key,
  title text not null,
  description text not null,
  "ImageUrl" text,
  "ReferenceUrl" text,
  category text not null check (category in ('low', 'mid', 'high')),
  created_at timestamptz not null default now()
);

alter table public."Blog" enable row level security;

grant usage on schema public to anon, authenticated;
grant select, insert on table public."Blog" to anon, authenticated;

drop policy if exists "allow_public_read_blog" on public."Blog";
create policy "allow_public_read_blog"
on public."Blog"
for select
to anon, authenticated
using (true);

drop policy if exists "allow_anonymous_insert_blog" on public."Blog";
create policy "allow_anonymous_insert_blog"
on public."Blog"
for insert
to anon, authenticated
with check (true);
```

## 2) Lấy thông tin Project URL và anon key

Vào **Project Settings** → **API**:
- `Project URL`
- `anon public` key

## 3) Cấu hình local (`.env`)

Trong `prism.landing_page`, tạo file `.env` (hoặc copy từ `.env.example`) và điền:

```bash
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key
```

Sau đó chạy lại app:

```bash
npm run dev
```

## 4) Cấu hình trên Vercel

Vào project trên Vercel → **Settings** → **Environment Variables** và thêm:
- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_ANON_KEY`

Chọn các môi trường cần áp dụng (`Production`, `Preview`, `Development`) rồi **Redeploy**.

## 5) Kiểm tra hoạt động

1. Mở trang test, nhập `Tên/Tuổi/Email`.
2. Làm xong bài test.
3. Ở màn kết quả sẽ có trạng thái:
   - `Đang lưu kết quả của bạn...`
   - `Đã lưu kết quả thành công.`
4. Vào Supabase → **Table Editor** → `test_submissions` để thấy dữ liệu mới.

## 6) Nếu vẫn không lưu được

Trang `Test` hiện sẽ hiển thị chi tiết lỗi Supabase ngay bên dưới nút `Làm lại`.

Các lỗi thường gặp:
- `new row violates row-level security policy`: chưa tạo đúng policy hoặc policy chưa áp dụng cho `anon`.
- `permission denied for table test_submissions`: thiếu `grant insert` cho `anon`/`authenticated`.
- `violates check constraint`: cấu trúc bảng cũ khác schema hiện tại (ví dụ `result_class`).
- `net::ERR_NAME_NOT_RESOLVED`: `VITE_SUPABASE_URL` sai Project URL (domain không tồn tại).

Với lỗi `ERR_NAME_NOT_RESOLVED`, kiểm tra nhanh:
1. Vào **Project Settings** → **API** → copy lại đúng `Project URL`.
2. Đảm bảo URL có dạng `https://<project-ref>.supabase.co` (không phải `.com`, không thiếu/ký tự dư).
3. Dán lại vào `.env`, rồi **tắt và chạy lại** `npm run dev`.

Nếu bảng đã tạo từ trước, nên kiểm tra lại schema bằng SQL sau:

```sql
select column_name, data_type
from information_schema.columns
where table_schema = 'public' and table_name = 'test_submissions'
order by ordinal_position;
```

- `email`
- `score`
- `result_class`
- `result_title`
- `created_at`
